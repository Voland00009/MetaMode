# ADR 0001: Mem0 cloud (free tier) как провайдер памяти для Phase 1

**Status:** accepted
**Date:** 2026-04-08

## Context

MetaMode Phase 1 = persistent memory layer поверх Claude Code. Нужен провайдер, который:
- предоставляет save/retrieve с семантическим поиском из коробки,
- не требует ручного управления векторной БД,
- позволяет быстро проверить продуктовую гипотезу "zero-ceremony memory через hooks",
- не тащит за собой Docker/self-host ceremony на старте.

Гипотеза, которую Phase 1 проверяет: hooks-driven автоматическая память через Claude Code lifecycle даёт реальную пользу без вовлечения юзера. Если гипотеза провалится, никакая инфраструктура не спасёт. Если подтвердится — всегда можно мигрировать на более тяжёлое решение.

Полный landscape-обзор — в `~/.claude/plans/metamode-research-synthesis.md` (5 кандидатов сравнены по функциям, лицензиям, зрелости, цене).

## Alternatives considered

1. **Graphiti** — мощный graph-based memory, но требует Neo4j, высокий порог входа, ceremony противоречит fail-fast подходу. Отвергнут.
2. **Cognee** — интересен, но менее зрелый, меньше community, API менее стабилен. Отвергнут как более рискованный.
3. **Supermemory** — SaaS с нечётким позиционированием, менее явный фокус на agent memory, меньше adoption. Отвергнут окончательно (Q4).
4. **claude-mem** — маргинальный проект, лицензия/safety не проверены, adoption низкий. Отвергнут без дальнейшей проверки (Q3).
5. **Self-host Mem0 (Docker)** — тот же SDK, но с docker-compose и локальной векторной БД. Отвергнут на Phase 1: добавляет инфра-ceremony, которая не проверяет продуктовую гипотезу. Остаётся кандидатом на Phase 2, если cloud провалится.
6. **Mem0 cloud free tier — выбранный вариант.**

## Decision

Мы будем использовать **Mem0 cloud (free tier)** как провайдер памяти для MetaMode Phase 1.

## Consequences

**Позитивные:**
- Минимум setup: API key → SDK → работает
- Семантический поиск, метадата-фильтры, update/delete API из коробки
- Фокусируемся на продуктовой гипотезе (hooks-driven memory), а не на инфраструктуре
- Free tier покрывает объёмы соло-разработчика

**Негативные:**
- Зависимость от внешнего SaaS (доступность, цены, политика)
- Данные живут у вендора — не годится для проектов с sensitive данными (MetaMode — персональный tooling, ок)
- Free tier имеет лимиты, которые могут упереться — потребуется миграция
- Lock-in на Mem0 SDK; если захочется сменить провайдера — нужна прослойка (отложено до Phase 2)

**Нейтральные:**
- Решение сознательно reversible: self-host Mem0 использует тот же SDK, миграция одним env var
- Если гипотеза zero-ceremony провалится — провайдер становится нерелевантен, и ADR уходит в `superseded`
