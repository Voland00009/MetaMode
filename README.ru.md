# MetaMode

**Персистентная wiki-память для Claude Code.** Сессии захватываются автоматически, компилируются в структурированные wiki-статьи и внедряются при старте сессии. Claude помнит.

Форк [coleam00/claude-memory-compiler](https://github.com/coleam00/claude-memory-compiler) с аудитом качества, мгновенным сохранением, RAW-инбоксом и интеграцией с Agent SDK.

---

## Проблема

Claude Code забывает всё между сессиями. Каждый новый разговор начинается с нуля — вы заново объясняете свой стек, повторяете прошлые решения и теряете уроки, которые уже усвоили. Встроенная auto-memory помогает, но она поверхностная: хранит факты, а не структурированные знания.

## Решение

MetaMode автоматически захватывает ваши сессии через hooks, извлекает важное, фильтрует шум двухпроходным аудитом качества и компилирует всё в структурированную wiki. При старте каждой сессии wiki-индекс и свежий контекст внедряются автоматически — Claude начинает работу, уже зная, что было раньше.

## Как это работает

```
┌─────────────────┐     ┌───────────┐     ┌───────────┐     ┌───────────────┐
│  Вы работаете    │────▶│  Hooks    │────▶│ flush.py  │────▶│  Дневной лог  │
│  в Claude Code   │     │  захват   │     │ извлечение│     │  (дополнение) │
└─────────────────┘     │  сессии   │     │ + аудит   │     └───────┬───────┘
                        └───────────┘     └───────────┘             │
                                                                    ▼
┌─────────────────┐     ┌───────────┐     ┌─────────────────────────┐
│  Следующая       │◀────│  Внедрение│◀────│  Wiki-статьи            │
│  сессия с        │     │  при      │     │  (compile.py)           │
│  контекстом      │     │  старте   │     └─────────────────────────┘
└─────────────────┘     └───────────┘
```

**Конец сессии** → hooks извлекают транскрипт → `flush.py` суммирует через Agent SDK → аудит качества фильтрует мусор → запись в дневной лог.

**Старт сессии** → hook читает wiki-индекс + свежий дневной лог → Claude видит контекст сразу.

**Компиляция** → дневные логи → структурированные wiki-статьи с перекрёстными ссылками.

## До и после

| Без MetaMode | С MetaMode |
|---|---|
| «Какой фреймворк мы используем?» | Claude уже знает ваш стек |
| «Помнишь то решение по авторизации на прошлой неделе?» | Решение в wiki, внедряется при старте |
| «Мы уже пробовали X, и это не сработало, потому что...» | Урок сохранён, Claude не повторит ошибку |
| Сессия каждый раз начинается с чистого листа | Сессия стартует с полным контекстом за <5 секунд |

## Быстрый старт

```bash
git clone https://github.com/Voland00009/MetaMode.git ~/Dev/MetaMode
cd MetaMode && uv sync
```

Затем настройте hooks в `~/.claude/settings.json`:

```json
{
  "hooks": {
    "SessionStart": [{"type": "command", "command": "uv run --directory ~/Dev/MetaMode python hooks/session_start.py"}],
    "SessionEnd": [{"type": "command", "command": "uv run --directory ~/Dev/MetaMode python hooks/session_end.py"}],
    "PreCompact": [{"type": "command", "command": "uv run --directory ~/Dev/MetaMode python hooks/pre_compact.py"}],
    "UserPromptSubmit": [{"type": "command", "command": "uv run --directory ~/Dev/MetaMode python hooks/user_prompt_submit.py"}]
  }
}
```

> Замените `~/Dev/MetaMode` на реальный путь к вашему клону. На Windows: `C:/Users/YOU/Dev/MetaMode`

Полная инструкция по установке с проверкой и решением проблем — в [docs/setup.md](docs/setup.md).

## Что вы получаете

| Инструмент | Что делает | Как |
|------|-------------|-----|
| **Auto-capture** | Сессии сохраняются автоматически | Hooks срабатывают при завершении сессии / compact |
| **`!save <текст>`** | Мгновенная заметка, 0 токенов | Введите в чат — блокирует prompt, сохраняет напрямую |
| **compile.py** | Дневные логи → wiki-статьи | `uv run python scripts/compile.py` |
| **query.py** | Запрос к wiki из командной строки | `uv run python scripts/query.py "вопрос"` |
| **ingest_raw.py** | Внешние документы → wiki | Положите в `raw/`, затем запустите скрипт |
| **lint.py** | 7 проверок здоровья | `uv run python scripts/lint.py` |
| **`/reflect`** | Рефлексия в конце сессии | 4 структурированных вопроса → дневной лог |

## Ключевые отличия от оригинала coleam00

1. **Agent SDK** вместо `claude -p` subprocess — чистый API, нет ограничений на длину аргументов CLI, те же $0 на подписке Max
2. **Аудит качества** — двухпроходный LLM-пайплайн в `flush.py`: Pass 1 извлекает, Pass 2 фильтрует мусор до попадания в дневные логи
3. **Хук `!save`** — введите `!save <текст>` в Claude Code, чтобы мгновенно сохранить мысль; блокирует prompt (0 токенов)
4. **RAW-инбокс** — `ingest_raw.py` обрабатывает внешние документы (статьи, заметки со встреч, исследования) наряду с захватом разговоров
5. **Отслеживание затрат** — `scripts/state.json` накапливает общую стоимость Agent SDK по всем операциям
6. **Авто-компиляция** — `flush.py` автоматически запускает `compile.py` после 18:00, если дневные логи изменились
7. **Категоризация статей** — теги и поле категории в схеме wiki-статей для структурированной организации знаний

## Структура проекта

```
MetaMode/
├── hooks/                  # Lifecycle-хуки Claude Code
│   ├── session_start.py        # Внедряет wiki-контекст при старте сессии
│   ├── session_end.py          # Захватывает транскрипт при закрытии сессии
│   ├── pre_compact.py          # Захватывает перед compaction контекста
│   ├── user_prompt_submit.py   # Перехватчик !save
│   └── shared.py               # Общие утилиты для хуков
├── scripts/                # Основной пайплайн
│   ├── flush.py                # Транскрипт → дневной лог (2 прохода с аудитом)
│   ├── compile.py              # Дневные логи → wiki-статьи
│   ├── ingest_raw.py           # Внешние документы → wiki-статьи
│   ├── lint.py                 # 7 структурных + LLM-проверок здоровья
│   ├── memory_lint.py          # Проверки auto-memory Claude Code
│   ├── query.py                # CLI-интерфейс запросов к wiki
│   ├── config.py               # Пути и константы
│   └── utils.py                # Общие утилиты
├── daily/                  # Дневные логи сессий (неизменяемые, append-only)
├── knowledge/              # Скомпилированная wiki
│   ├── index.md                # Главная навигационная таблица
│   ├── log.md                  # Журнал операций
│   ├── concepts/               # Атомарные статьи-знания
│   ├── connections/            # Связи между концепциями
│   └── qa/                     # Сохранённые ответы на вопросы
├── raw/                    # Инбокс для внешних документов
├── reports/                # Отчёты lint
├── AGENTS.md               # Схема wiki-статей для Agent SDK
├── CLAUDE.md               # Инструкции проекта для Claude Code
└── pyproject.toml          # Python-зависимости
```
