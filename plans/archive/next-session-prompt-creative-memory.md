# Жёсткие требования к финальной системе

1. **Auto-save через хуки** (zero ceremony). Пользователь прямо сказал: «Я буду забывать, если у нас не будет так, как у парня в видео». Ручные команды — провал.
2. **Self-learning** — накапливать уроки из сессий, не повторять ошибок. Главная ценность системы.
3. **Token economy** — не раздувать контекст при SessionStart. Lazy retrieval > жадная загрузка. Экономия токенов — явная цель, а не побочный эффект.
4. **Cost минимальный** — целевой диапазон <50 USD/mo совокупно по подпискам. Идеально: работать на текущей Anthropic подписке (через Claude Agent SDK headless) без дополнительных SaaS. «Никому не нужен вариант +200 USD/mo».
5. **Transparency** — пользователь должен иметь возможность **посмотреть глазами**, что сохранилось. Чёрный ящик Mem0 dashboard — минус, открытые markdown — плюс.
6. **Moat-awareness** — Anthropic скоро выкатит свою память. Наша система должна быть либо умнее (self-learning, cross-project, компилируемая БЗ), либо нишевее (конкретный workflow), либо и то и другое. Быть просто «ещё одним вариантом» — бессмысленно.
7. **Portable** — данные в файлах или легко экспортируемы, минимум vendor lock-in.

## Фаза C — Предложить 3 нестандартных архитектуры (твоя настоящая работа)

Это самая важная фаза. Здесь ты должен **перестать сравнивать готовое** и начать **изобретать**. Опираясь на инвентарь из фазы A, элементы Карпати из фазы B и существующие наработки MetaMode, предложи **три разные архитектуры** финальной системы памяти. Они должны отличаться **принципиально**, а не параметрами.

Примеры направлений (используй как искры, не как готовые варианты):

- **Trilevel hot/warm/cold** с UserPromptSubmit hook как query router
- **Compile-on-idle через scheduled agent** вместо compile в Stop hook
- **Self-learning через PostToolUse**: анализировать реальные действия агента, а не только сессии
- **Lazy hybrid**: markdown как storage + локальный cheap semantic index (SQLite FTS5 или embeddings локально) как accelerator
- **Cross-project knowledge mesh**: символические ссылки между CLAUDE.md разных проектов
- **Compressed daily-log через Agent SDK headless на подписке** — нулевая стоимость
- **Query-time retrieval через skill-trigger**, не жадный SessionStart
- **Что-то, чего ещё никто не пробовал** — и это самая ценная часть задания

Каждая архитектура должна быть описана так, чтобы я (пользователь) мог оценить:

1. **Поток данных** — что, куда, когда, каким хуком / скриптом / агентом
2. **Token budget** — сколько токенов тратится на запись, на recall, в какой момент
3. **Стоимость** — какие подписки / API ключи нужны, месячный чек
4. **Self-learning механизм** — как именно система учится, а не просто пишет логи
5. **Прозрачность** — где пользователь может посмотреть содержимое
6. **Что оригинально** — чем отличается от Mem0, от Karpathy, от того что выкатит Anthropic
7. **Риски и unknown-unknowns** — где может сломаться

**Выход фазы C:** три пронумерованные архитектуры с этими семью пунктами каждая. Без рекомендации пока — пользователь сначала хочет увидеть варианты, а потом обсуждать.
