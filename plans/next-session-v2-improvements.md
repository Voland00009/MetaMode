## Задача 1: Объективная проверка оставшихся 5 добавлений

### Что проверить:

**1. Pending review (качество vs скорость)**
- Утверждение: "предотвращает мусор" (95% мусора в mem0)
- Контраргумент: mem0 — другой инструмент. Может, coleam00 flush.py извлекает лучше, и review не нужен?
- **Как проверить:** Посмотреть `scripts/pending-review.md` и daily logs — сколько FLUSH_OK, сколько approval, сколько reject? Есть ли данные за реальное использование?

**2. Глобальные hooks vs per-project**
- Утверждение: "одна wiki для всех проектов = лучше"
- Контраргумент: знания о React-проекте не нужны в Python-проекте
- **Как проверить:** Подготовить structured prompt для Gemini/ChatGPT: "one global wiki vs per-project wikis for a solo developer with 3-5 projects?"

**3. Categorization tags**
- Утверждение: "лучшая организация"
- Контраргумент: Karpathy 100+ статей без категоризации, index.md хватает
- **Как проверить:** Сколько у нас статей сейчас? При <50 это premature optimization?

**4. !save hook**
- Утверждение: "полезно для quick-capture"
- Контраргумент: Если SessionEnd flush работает хорошо, зачем ручной capture?
- **Как проверить:** Сколько раз !save реально использовался? Есть ли в daily logs записи от !save?

**5. Enriched session start**
- Утверждение: "больше контекста = лучше"
- Контраргумент: Больше токенов = медленнее старт. Нужна ли compile reminder если <20 статей?

### Метод:
- Собрать фактические данные из daily logs и pending-review.md
- Для субъективных вопросов (global vs per-project) — подготовить промпты для мультимодельной верификации через подписки (Gemini PRO, ChatGPT Plus, Perplexity Pro)

## Задача 3: Автоматизация — чтобы не нужно было помнить

Пользователь говорил: "я не уверен, что понимаю, как правильно сохранять данные".

### Текущий workflow:
- ✅ SessionEnd → flush → pending review (авто)
- ✅ PreCompact → flush (авто)
- ✅ SessionStart → inject index + pending review (авто)
- ✅ !save → instant capture (ручной)
- ❓ Compile → после 18:00 автоматически ИЛИ ручной
- ❌ Approve pending review → ручной ("approve")
- ❌ Process RAW → ручной ("обработай RAW")

### Что проверить и улучшить:
1. **Auto-compile после 18:00 реально работает?** — есть evidence в логах?
2. **Pending review: что если пользователь никогда не говорит "approve"?** — данные копятся навечно. Auto-approve через N дней?
3. **Написать шпаргалку** (1 страница) — что автоматически, что руками
4. **RAW reminder** — работает ли? Достаточно ли напоминания или нужен auto-ingest?

### Идеи для обсуждения (не делать без одобрения):
- Auto-approve через 3 дня если не отклонено?
- Периодический compile по крону?
- Auto-ingest RAW при SessionStart?
