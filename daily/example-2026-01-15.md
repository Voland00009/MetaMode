### Session: REST API Authentication Setup (14:22)

**Context:** Setting up JWT authentication for a FastAPI project, choosing between PyJWT and python-jose libraries.

**Key Exchanges:**
- Compared PyJWT vs python-jose: PyJWT is simpler and better maintained, python-jose hasn't been updated since 2023
- Discussed token refresh strategy: decided on short-lived access tokens (15min) + long-lived refresh tokens (7 days)
- Explored storing refresh tokens in httponly cookies vs localStorage — cookies are more secure against XSS

**Decisions Made:**
- Use PyJWT for token handling — simpler API, active maintenance, no extra dependencies
- Access token: 15 min expiry, refresh token: 7 days, stored in httponly cookie
- Token blacklist via Redis SET with TTL matching token expiry — no need for periodic cleanup

**Lessons Learned:**
- Python `datetime.utcnow()` is deprecated in 3.12+ — use `datetime.now(timezone.utc)` instead
- FastAPI's `Depends()` system makes auth middleware clean: one dependency function, inject everywhere
- Context managers (`with` statement) are ideal for database session handling — ensures cleanup even on exceptions
