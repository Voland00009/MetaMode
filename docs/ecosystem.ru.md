# Экосистема MetaMode

🌐 [English](ecosystem.md) | **Русский**

Как все части MetaMode работают вместе, чтобы давать Claude постоянную память между сессиями, проектами и внешними источниками.

## Проблема

Claude Code stateless — каждая сессия начинается с нуля. Вы объясняете один и тот же контекст, заново пересказываете принятые решения, повторно открываете те же баги. MetaMode решает это pipeline'ом, который автоматически захватывает, компилирует и инжектит знания.

Но разговоры — лишь один источник знаний. Вы также учитесь из статей, документации, исследований и чужого кода. Экосистема объединяет всё это в единую память.

## Компоненты

```text
Экосистема MetaMode
├── Hooks          — авто-захват разговоров → daily logs
├── Compile        — daily logs → структурированные wiki-статьи
├── RAW Inbox      — внешние документы → wiki-статьи
├── Web Clipper    — веб-страницы → RAW inbox (один клик)
├── Obsidian       — просмотр, поиск, визуализация wiki
└── NotebookLM     — аудио-обзоры, квизы, чат со знаниями
```

### Hooks (авто-захват)

Три lifecycle-хука Claude Code срабатывают автоматически из любого проекта:

- **SessionStart** — инжектит index wiki + последний daily log в контекст Claude
- **SessionEnd / PreCompact** — извлекает транскрипт разговора, суммаризирует его через Agent SDK и пишет запись в daily log

Вам не нужно делать ничего. Открыл Claude Code, поработал, закрыл — хуки сделают остальное.

### Compile (daily logs → wiki)

`scripts/compile.py` читает записи daily log и превращает их в структурированные wiki-статьи в `knowledge/concepts/` и `knowledge/connections/`. Каждая статья атомарна — один концепт на файл, wiki-ссылки соединяют связанные идеи.

Compile запускается автоматически после 18:00, когда flush обнаруживает новые логи, или вручную — `uv run python scripts/compile.py`.

### RAW Inbox (внешние документы)

Положите любой `.md` или `.txt` файл в `raw/`. Запустите `uv run python scripts/ingest_raw.py`. Скрипт создаст wiki-статьи и переместит обработанные файлы в `raw/processed/`.

Это точка входа для знаний извне Claude — заметки, транскрипты, скопированная документация, всё что угодно в markdown.

### Web Clipper (веб → RAW)

Obsidian Web Clipper — расширение браузера, которое сохраняет любую веб-страницу как markdown-файл прямо в `raw/`. Один клик чтобы клипнуть, одна команда чтобы ингестировать.

```text
Пост в блоге → Web Clipper → raw/blog-post.md → ingest_raw.py → knowledge/concepts/blog-insight.md
```

Это превращает браузер в инструмент захвата знаний — статьи, документация, ответы Stack Overflow, всё что вы читаете может стать частью памяти Claude.

См. [Web Clipper setup](web-clipper-setup.ru.md) для установки.

### Obsidian (визуализация)

Директория `knowledge/` MetaMode — это нативный vault Obsidian. Откройте её и получите:

- **Graph view** — смотрите как концепты соединяются между проектами и темами
- **Полнотекстовый поиск** — находите что угодно в daily logs, wiki и RAW inbox
- **Backlinks** — обнаруживайте какие статьи ссылаются на данный концепт
- **Прямое редактирование** — правьте или расширяйте wiki-статьи в удобном редакторе

Obsidian опционален — wiki это plain markdown под git. Но graph view делает видимыми паттерны, которые вы никогда не найдёте, читая файлы по одному.

См. [Obsidian setup](obsidian-setup.ru.md) для настройки.

### NotebookLM (обучение и повторение)

Google NotebookLM умеет ингестировать ваши wiki-статьи и генерировать:

- **Аудио-обзоры** — подкаст-стиль саммари, который можно слушать на прогулке
- **Квизы** — проверка памяти на концепты, изученные недели назад
- **Чат** — задавать вопросы по своей же базе знаний без расхода токенов Claude

NotebookLM лучше всего подходит для повторения и консолидации — он помогает внутренне усвоить знания, захваченные MetaMode.

См. [NotebookLM setup](notebooklm-setup.ru.md) для настройки.

## Что вы получаете из коробки

После клонирования MetaMode и настройки хуков:

| Фича | Нужна настройка | Постоянные усилия |
|------|-----------------|-------------------|
| Авто-захват сессий | Настроить хуки один раз | Никаких — автоматически |
| Compile wiki | Не нужно | Запускается после 18:00 или вручную |
| RAW inbox | Не нужно | Положить файлы, запустить ingest |
| Web Clipper | Установить расширение браузера | Один клик на страницу |
| Obsidian | Открыть папку как vault | Никаких — читает markdown |
| NotebookLM | Настроить CLI + cookie-аутентификация | Загружать источники периодически |

## Почему вместе > по отдельности

Каждый инструмент полезен сам по себе. Вместе они создают петлю обратной связи:

1. Вы **узнали** что-то в сессии Claude → хуки захватили
2. Вы **прочитали** статью, расширяющую тему → Web Clipper захватил
3. Оба **компилируются** в один wiki → связи формируются автоматически
4. Вы **видите** связи в графе Obsidian → новые инсайты
5. Вы **повторяете** через аудио NotebookLM → лучше запоминаете
6. Следующая сессия Claude **стартует со всем этим** → лучшие ответы → вы учитесь больше

Wiki — это хаб. Всё течёт в него, и всё читает из него.
