# Интеграция с Obsidian

🌐 [English](obsidian-setup.md) | **Русский**

Директория `knowledge/` MetaMode — это нативный vault Obsidian. Никаких плагинов или расширений не нужно — Obsidian читает markdown и wiki-ссылки (`[[concept]]`) из коробки.

## Почему это важно

MetaMode строит wiki всего, что вы узнаёте по всем проектам — но markdown-файлы в директории трудно исследовать. Obsidian превращает эту директорию в визуальную, searchable базу знаний:

- **Graph view** показывает связи между концептами разных проектов и сессий — паттерны, которые никогда не увидишь, читая файлы по одному
- **Моментальный поиск** по всем daily logs, wiki-статьям и raw inbox — быстрее grep, с превью
- **Прямое редактирование** — меняйте wiki-статьи в Obsidian, MetaMode подхватит изменения (это те же файлы на диске)

Obsidian опционален — MetaMode работает полностью без него. Но без Obsidian вы теряете визуальный слой, который делает накопленные знания реально discoverable.

## Установка

1. [Скачать Obsidian](https://obsidian.md/) (бесплатно для личного использования)
2. Открыть как vault: **File → Open Vault → Open folder as vault** → выбрать корневую директорию `MetaMode/`
3. Готово. Все wiki-статьи, daily logs и связи сразу доступны для просмотра.

## Что вы получаете

### Graph view
Откройте Graph View (`Ctrl+G` / `Cmd+G`), чтобы увидеть как ваши знания связаны. Статьи MetaMode используют `[[wiki-ссылки]]`, которые Obsidian рендерит как кликабельные соединения. Кластеры показывают, какие концепты относятся друг к другу.

### Live preview
Редактируйте `knowledge/concepts/*.md` напрямую — изменения сохраняются на диск и будут подхвачены compile и lint pipelines MetaMode.

### Поиск
`Ctrl+Shift+F` ищет по всем daily logs, wiki-статьям и raw inbox. Быстрее grep для исследования базы знаний.

### Daily Notes
Директория `daily/` MetaMode содержит автоматически сгенерированные daily logs. Откройте любой `YYYY-MM-DD.md`, чтобы посмотреть, что Claude захватил за день.

## Use cases

### 1. Визуальное исследование знаний
После нескольких недель использования MetaMode ваш graph показывает сеть связанных концептов. Кликайте по узлам, чтобы вновь открывать паттерны, изученные недели назад. Особенно полезно перед началом работы в области, к которой давно не возвращались.

### 2. Быстрые ручные заметки
Откройте любой daily log в Obsidian и печатайте напрямую. Compile MetaMode подхватит ваши дополнения при следующем запуске. Также можно использовать `!save <текст>` в Claude Code — мгновенное сохранение за 0 токенов.

### 3. Повторение перед сессиями
Перед сессией Claude Code просмотрите graph или последние daily logs в Obsidian. Это даст контекст, который поможет задавать лучшие вопросы и избегать повторений прошлых ошибок.

### 4. Аудит знаний
Сортируйте статьи по дате изменения, находите концепты без входящих ссылок (orphans) или слишком короткие статьи. `lint.py` MetaMode делает это автоматически, но Obsidian делает визуально.

### 5. Кросс-проектные паттерны
Поскольку MetaMode захватывает знания из ВСЕХ проектов (глобальные хуки), graph Obsidian показывает паттерны между проектами — например, одна и та же техника дебаггинга появляется и в Python-бэкенде, и в React-фронтенде.

## Важные замечания

- **Obsidian опционален.** MetaMode работает полностью без него — wiki это plain markdown-файлы под git.
- **Никаких кастомных плагинов не нужно.** Всё работает с ванильным Obsidian.
- **Не переименовывайте файлы в Obsidian**, если хотите, чтобы wiki-ссылки остались консистентными. Для структурных изменений используйте compile MetaMode.
- **Vault — это корень MetaMode**, а не только `knowledge/`. Это даёт видеть daily logs, raw inbox и wiki-статьи в одном месте.

## Web Clipper

Obsidian Web Clipper сохраняет веб-страницы прямо в RAW inbox MetaMode. Один клик чтобы клипнуть, одна команда чтобы ингестировать — Claude запомнит, что вы читали.

См. [Web Clipper setup](web-clipper-setup.ru.md) для установки и настройки.
