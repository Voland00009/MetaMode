# Возможности MetaMode

Полная документация работы MetaMode — от захвата сессий до извлечения знаний из wiki.

## Обзор пайплайна

MetaMode имеет два пайплайна:

1. **Захват диалогов** — автоматический, работает через hooks при каждой сессии Claude Code
2. **Внешний импорт** — ручной, для статей/транскриптов/заметок извне Claude Code

Оба пайплайна дают одинаковый результат: структурированные wiki-статьи в `knowledge/`.

---

## 1. Пайплайн захвата диалогов

### Hook: SessionStart (`hooks/session_start.py`)

**Когда:** При каждом запуске сессии Claude Code, в любом проекте (hooks глобальные).

**Что делает:**
- Читает `knowledge/index.md` — полное оглавление wiki
- Получает последние 30 строк из сегодняшнего или вчерашнего дневного лога
- Проверяет наличие нескомпилированных дневных логов (>3 файлов = напоминание о compile)
- Проверяет наличие необработанных файлов в RAW inbox (`raw/`)
- Выводит JSON с `additionalContext` — Claude видит это сразу

**Бюджет размера:** `MAX_CONTEXT_CHARS = 20,000` — обрезает, если индекс wiki становится большим.

**Результат:** Claude начинает сессию, зная содержимое wiki и что произошло недавно, без сканирования файлов.

### Hook: SessionEnd (`hooks/session_end.py`)

**Когда:** Сессия Claude Code завершается (пользователь закрывает, `/exit`, закрытие терминала).

**Что делает:**
- Читает JSONL-транскрипт из файла транскриптов Claude
- Извлекает последние 30 реплик в виде markdown (пары user/assistant)
- Записывает контекст во временный файл в `scripts/`
- Запускает `flush.py` как фоновый процесс (неблокирующий, <1с)

**Нюансы:**
- Защита от рекурсии: если установлена переменная окружения `CLAUDE_INVOKED_BY`, hook сразу завершается (предотвращает цикл flush.py → Agent SDK → Claude Code → hook)
- Windows: использует флаг `CREATE_NO_WINDOW` для предотвращения мигания консоли
- Максимум контекста: 15,000 символов, обрезка по границам реплик

### Hook: PreCompact (`hooks/pre_compact.py`)

**Когда:** Claude Code собирается выполнить auto-compact (контекстное окно заполнено).

**Что делает:** То же, что SessionEnd — извлекает транскрипт и запускает flush.py. Это страховка: без неё решения из начала сессии потерялись бы при сжатии контекста.

**Ключевое отличие:** `MIN_TURNS_TO_FLUSH = 5` (против 1 для SessionEnd) — избегает flush для крошечных compaction.

### Hook: UserPromptSubmit (`hooks/user_prompt_submit.py`)

**Когда:** Пользователь вводит `!save <текст>` в Claude Code.

**Что делает:**
- Разбирает текст после `!save`
- Записывает напрямую в сегодняшний дневной лог как запись "Quick Save"
- Завершается с кодом 2 — **блокирует отправку промпта в Claude** (0 токенов потрачено)
- Показывает подтверждение через stderr

**Сценарий использования:** Зафиксировать мысль посреди сессии без траты токенов. Пример: `!save Decision: we're using PostgreSQL for the auth service because SQLite can't handle concurrent writes`

### Скрипт: flush.py

**Когда:** Запускается hooks SessionEnd или PreCompact.

**Что делает (2 прохода):**

**Pass 1 — Извлечение:**
- Читает временный файл контекста
- Отправляет в Claude Agent SDK со структурированным промптом
- Claude извлекает: Context, Key Exchanges, Decisions, Lessons Learned, Action Items
- Если нечего сохранять → записывает `FLUSH_OK` в дневной лог

**Pass 2 — Quality Audit:**
- Отправляет извлечённый контент в ещё один вызов Agent SDK
- Проверяет: стоит ли это хранить? (не просто "мы сделали X, потом Y")
- Консервативный подход: при сомнениях сохраняет контент
- Если помечен: оборачивает контент в HTML-комментарий `<!-- AUDIT_FLAG: reason -->`
- Помеченные записи остаются в дневном логе, но пропускаются при компиляции

**После обоих проходов:**
- Записывает в `daily/YYYY-MM-DD.md` с временной меткой
- Дедупликация: пропускает, если та же сессия выполняла flush в последние 60 секунд
- Проверяет, нужно ли запускать auto-compile (после 18:00, если логи изменились)

### Скрипт: compile.py

**Когда:** Вручную (`uv run python scripts/compile.py`) или автоматически из flush.py после 18:00.

**Что делает:**
- Читает дневные логи, которые ещё не скомпилированы (сравнение хешей)
- Отправляет каждый лог + схему AGENTS.md + существующую wiki в Agent SDK
- Agent SDK создаёт/обновляет wiki-статьи: concepts, connections, index, log
- Имеет файловые инструменты: Read, Write, Edit, Glob, Grep (до 30 итераций)
- Пропускает записи с audit-флагами
- Отслеживает состояние компиляции в `state.json`

**Опции:**
- `--all` — принудительная перекомпиляция всего
- `--file daily/2026-04-11.md` — компиляция конкретного файла
- `--dry-run` — показать, что будет скомпилировано

## 2. Пайплайн внешнего импорта

### Скрипт: ingest_raw.py

**Когда:** Вручную (`uv run python scripts/ingest_raw.py`) или когда пользователь говорит "обработай RAW".

**Вход:** Файлы `.md` или `.txt`, помещённые в директорию `raw/`.

**Что делает:**
- Тот же подход, что и compile.py, но для внешних документов
- Agent SDK читает документ + схему + существующую wiki
- Создаёт wiki-статьи, обновляет индекс
- Перемещает обработанные файлы в `raw/processed/`

**Сценарий использования:** Загрузка резюме статей, транскриптов встреч, исследовательских заметок — чего угодно извне сессий Claude Code.

## 3. Структура Wiki

### knowledge/index.md
Главная навигация. Каждая статья занимает одну строку:
```
| [[concepts/slug]] | One-line summary | source-file | date |
```
Claude читает это первым для навигации по wiki.

### knowledge/concepts/
Атомарные статьи знаний. Каждая содержит:
- YAML frontmatter (title, aliases, tags, category, sources, dates)
- Структуру Context → Problem → Lesson
- Разделы Key Points, Details, Related Concepts
- Wikilinks на другие статьи

### knowledge/connections/
Статьи о связях между концепциями. Для неочевидных связей между 2+ концепциями.

### knowledge/qa/
Сохранённые ответы из `query.py --file-back`. Вопрос + синтезированный ответ + источники.

### knowledge/log.md
Лог операций compile/ingest/lint с временными метками.

## 4. Проверки здоровья

### Скрипт: lint.py

7 проверок:
1. **Broken links** — wikilinks, указывающие на несуществующие статьи
2. **Orphan pages** — статьи, на которые ни одна другая статья не ссылается
3. **Orphan sources** — дневные логи, ещё не скомпилированные
4. **Stale articles** — исходный лог изменился после компиляции
5. **Missing backlinks** — A ссылается на B, но B не ссылается обратно на A
6. **Sparse articles** — менее 200 слов
7. **Contradictions** — LLM читает все статьи и ищет противоречия (дорого)

Используйте `--structural-only`, чтобы пропустить LLM-проверку на противоречия.

Отчёты сохраняются в `reports/lint-YYYY-MM-DD.md`.

## 5. Интерфейс запросов

### Скрипт: query.py

CLI-доступ к wiki без открытия Claude Code.

```bash
uv run python scripts/query.py "How should I handle auth?"
uv run python scripts/query.py "What patterns do I use?" --file-back
```

С `--file-back`: сохраняет ответ как Q&A-статью в `knowledge/qa/`.

## Советы и нюансы

### Дедупликация flush
Если вы быстро закроете и снова откроете Claude Code, одна и та же сессия может вызвать flush дважды. `flush.py` дедуплицирует по session ID + 60-секундному окну.

### Audit-флаги
Записи с audit-флагами (HTML-комментарии в дневных логах) никогда не удаляются — они остаются для ручного просмотра. Compile их пропускает. Если вы хотите включить помеченную запись, удалите HTML-комментарий вручную.

### Учёт стоимости
`scripts/state.json` отслеживает `total_cost` от вызовов Agent SDK. На Max-подписке это $0, но счётчик полезен для мониторинга использования.

### Особенности Windows
- Флаг `CREATE_NO_WINDOW` предотвращает мигание консоли при запуске фоновых процессов
- `sys.stdin.reconfigure(encoding="utf-8")` в hooks — Windows по умолчанию использует cp1251
- Парсинг JSON имеет fallback для неэкранированных обратных слешей в путях Windows

### Формат дневных логов
Дневные логи неизменяемы — flush дописывает, никогда не модифицирует. Каждая запись имеет заголовок `### Section (HH:MM)`. Это делает git diff чистыми, а компиляцию идемпотентной.

### Время компиляции
Автоматический запуск срабатывает только после 18:00 (настраивается через `COMPILE_AFTER_HOUR` в flush.py). До этого запускайте compile вручную, когда хотите свежие wiki-статьи.
