# MetaMode — Vision v2 (полная картина)

**Дата:** 2026-04-06
**Статус:** Промежуточный вариант, до критического анализа

## Что это

MetaMode — коллективная самообучающаяся менторская система поверх Claude Code + Superpowers.
Claude Code перестаёт быть "исполнителем команд" и становится наставником.

## 9 блоков системы

### 1. Менторство, а не послушание
CC ведёт к цели, а не просто делает что скажут. Если пользователь идёт не туда — скажет. Если есть путь лучше — предложит и объяснит почему. Если пользователь знает что делает — отходит в сторону, наблюдает, вмешивается только на развилках.

### 2. Живая база знаний (Knowledge Base)
- Позитивные кейсы: каждый раз, когда проблема решена — записывает что было, как решили, что сработало
- Ошибки и баги: запоминает, предупреждает заранее, подсказывает как не попасть в ситуацию
- При повторной проблеме — поднимает из базы, не ищет с нуля
- Хранение: удалённый репозиторий (GitHub), грамотный формат, по папкам и файлам

### 3. Приём внешней информации
Пользователь находит статью/пост/файл — кидает системе. Она:
- Обрабатывает и структурирует
- Раскладывает по нужным папкам
- Потом сама предлагает, когда видит подходящую задачу

### 4. Проактивная помощь из базы + интернет
- Сначала смотрит в свою базу — похожие кейсы, материалы, подводные камни
- Если в базе нет — ищет в интернете, продумывает
- Предлагает конкретные варианты, объясняет почему, советует лучший

### 5. Умные уточняющие вопросы (из Prompt Booster)
- Предлагает количество вопросов: 0, 1-5, 5-9, 10-15
- Вопросы с вариантами ответов (a/b/c/d/свой)
- Пользователь выбирает глубину

### 6. Авто-роли экспертов (1-5 ролей)
- Автоматический подбор по домену и сложности
- Не важно видит ли пользователь кто работает — важен КПД

### 7. INoT — внутренние дебаты перед ответом
- Виртуальные агенты спорят, критикуют, проверяют
- Всё внутри одного вызова (~58% экономия токенов по исследованию)
- Продуманный результат вместо первой мысли

### 8. Многопользовательская система
- Каждый пользователь учит систему, она — его
- Позитивные кейсы копятся от разных людей из разных сфер
- Новый пользователь получает накопленный опыт
- Равные права, каждый берёт что нужно и дополняет

### 9. Архитектура хранения — цепочка файлов
- Главный файл не перегружен — только указатели
- Система идёт по цепочке файл → файл и находит нужное
- Проще, чем гуглить однотипные проблемы каждый раз

## Входные материалы (все прочитаны)
- INoT.docx — научная статья (PromptCode, +8% точности, -58% токенов)
- Prompt Booster (PB00-PB05) — GPT-система с машиной состояний, ролями, протоколом вопросов
- Answer Inspector — GPT для улучшения черновиков через INoT-дебаты
- INoT Risk Hunter — GPT для поиска рисков через INoT
- cc_guide_ru, boris_tips_ru — полные гайды Claude Code
- prompt_engineering_guide_ru — техники промптинга
- Claude_Secret_Codes — 100 промпт-шорткатов
