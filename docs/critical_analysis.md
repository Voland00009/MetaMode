# MetaMode — Критический анализ

**Дата:** 2026-04-06
**Статус:** Завершён, принят пользователем

## Общая оценка: 7/10

Идея сильная, но ~40% описанного — "было бы круто" без технической возможности сейчас.
Фокус на реализуемом — получится полезная вещь. Всё сразу — утонешь и забросишь.

## Что РЕАЛЬНО сработает

### 1. Менторский протокол — ДА, 100%
Правильно написанный CLAUDE.md + skill. CC уже умеет так — нужно настроить. Самая ценная и простая часть.

### 2. Цепочка файлов / архитектура — ДА
CLAUDE.md → указатели → файлы. Уже так работает. Нужно грамотно организовать.

### 3. Умные вопросы — ДА, но проще чем в Prompt Booster
Не нужна машина состояний. CC работает в диалоге. Скилл который учит задавать вопросы — реально и полезно.

### 4. Авто-роли — ЧАСТИЧНО
Один Claude смотрит с разных сторон. Не настоящие агенты. Пользу даст, магии не будет.

## Что СОМНИТЕЛЬНО

### 5. INoT в Claude Code — БОЛЬШОЙ ВОПРОС
- Статья: одиночные задачи (код, математика, QA)
- CC: агентная среда с tools, субагентами, planning
- Риск: конфликт с extended thinking, расход контекста на PromptCode
- **Решение:** опциональный скилл, тест на реальных задачах, замер разницы

### 6. Живая база знаний с автозаписью — СЛОЖНЕЕ ЧЕМ КАЖЕТСЯ
- Как определить "успешно решено"? Нужен явный сигнал от пользователя
- Формат: бездумное копление → свалка. Нужна структура
- Размер: 100 кейсов по 500 слов = половина контекста
- **Решение:** ручной триггер, жёсткий лимит, индекс + краткие записи

## Что скорее НЕ сработает (сейчас)

### 7. Автоструктурирование внешних материалов — ПРОБЛЕМАТИЧНО
- Между сессиями CC не помнит
- Нужна таксономия, которая будет меняться
- **Реалистичнее:** полуавтоматический режим в диалоге

### 8. Проактивная помощь + интернет — ОГРАНИЧЕННО
- База: да, если маленькая и организованная
- WebSearch: не Google, результаты средние
- CC не имеет инициативы между сессиями

### 9. Многопользовательность — САМОЕ СЛОЖНОЕ
- Технически: общий GitHub-репо, каждый клонирует
- Практически: модерация, разные стеки, конфликты git, приватность
- **Решение:** отложить. Сначала для одного пользователя.

## Главное ограничение: КОНТЕКСТНОЕ ОКНО
Каждый файл = часть контекста. CLAUDE.md + rules + skills + memory + кейсы + материалы = контекст закончился до начала работы. Минимализм обязателен.
