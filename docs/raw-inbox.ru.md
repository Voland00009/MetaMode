# RAW Inbox — ингестия внешних знаний

🌐 [English](raw-inbox.md) | **Русский**

Директория `raw/` — это inbox MetaMode для внешних документов. Положите туда markdown- или текстовые файлы, запустите скрипт ингестии, и они превратятся в структурированные wiki-статьи рядом с вашими автоматически захваченными знаниями из сессий.

## Как это работает

```
1. Положить .md или .txt файлы в raw/
2. Запустить: uv run python scripts/ingest_raw.py
3. Файлы → wiki-статьи в knowledge/concepts/ и knowledge/connections/
4. Оригиналы переезжают в raw/processed/
5. knowledge/index.md обновляется автоматически
```

## Формат файла

Для лучших результатов используйте такую структуру:

```markdown
# Заголовок концепта

## Context
Где вы встретили это, почему это важно.

## Key Insight
Ключевая идея — что вы хотите запомнить.

## Example
Конкретный пример или сниппет кода.
```

Но подойдёт любой markdown — скрипт ингестии через Claude Agent SDK извлечёт структуру из чего угодно.

## Use cases

### 1. Заметки из статей/блогов
Прочитали интересную статью? Скопируйте ключевые мысли в `.md` файл, положите в `raw/`, ингестируйте. Инсайт останется в вашем wiki навсегда — searchable, связанный с родственными концептами.

### 2. Заметки со встреч
После встречи запишите решения и action items в markdown. Ингестируйте в MetaMode — теперь Claude знает что было решено, без ваших повторений.

### 3. Хайлайты курсов/туториалов
Проходите онлайн-курс? Сохраняйте ключевые тейкэвэи по модулям. После ингестии можете спрашивать wiki: «что я узнал про индексы базы данных?»

### 4. Инсайты из code review
Нашли крутой паттерн в чужом code review? Сохраните в raw и ингестируйте. Станет wiki-статьёй, к которой можно вернуться через месяцы.

### 5. Дампы исследований
Работаете над проблемой? Складывайте ссылки, заметки и находки в raw. После ингестии Claude имеет контекст вашего исследования, без объяснений каждую сессию.

### 6. Внешняя документация
Скопируйте релевантные разделы документации (гайды фреймворков, API, внутренние wiki) в raw. Это даёт Claude Code постоянный доступ к информации, которой может не быть в обучающих данных.

## Почему это важно

Claude Code стартует с нуля каждую сессию. Без MetaMode вам пришлось бы заново объяснять контекст. RAW inbox позволяет скармливать Claude внешние знания, которые:

- **Переживают сессии** — сохранены в git, доступны навсегда
- **Компилируются** — сырые заметки становятся структурированными wiki-статьями с wiki-ссылками
- **Соединяются с знаниями из сессий** — compile связывает статьи из raw с статьями из разговоров
- **Не стоят ничего** — вызовы Agent SDK для ингестии покрыты Max-подпиской

## Напоминание в session start

MetaMode автоматически проверяет `raw/` на старте сессии. Если есть необработанные файлы, вы увидите:

```
3 unprocessed file(s) in raw/: notes.md, research.md, review.md.
Say 'обработай RAW' or run `uv run python scripts/ingest_raw.py`.
```

## Советы

- **Один концепт на файл** — получаются более чистые wiki-статьи, чем когда всё свалено в один файл
- **Используйте описательные имена** — `python-import-binding.md` лучше, чем `notes.md`
- **Не гонитесь за идеалом** — скрипт ингестии извлекает структуру даже из грубых заметок
- **Проверяйте результат** — откройте `knowledge/index.md` чтобы убедиться, что новые статьи выглядят правильно
- **Git отслеживает всё** — если ингестия выдала плохой результат, `git checkout` откатит
