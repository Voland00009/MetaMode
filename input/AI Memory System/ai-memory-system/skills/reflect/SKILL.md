# /reflect — Self-Improving Loop

После работы над задачей агент анализирует сессию и записывает уроки в два места:
1. **Knowledge Base** (Obsidian vault) — глобальная база, доступна во всех проектах
2. **CLAUDE.md проекта** — краткие lab-notes для быстрого контекста

## Шаг 1: Анализ сессии

Ответь себе на 4 вопроса:

1. **Скорость:** были ли лишние шаги? Можно ли было прийти к результату быстрее?
2. **Ошибки:** были ли неправильные подходы, откаты? Что их вызвало?
3. **Capabilities:** были ли моменты когда агент не знал что может что-то сделать?
4. **Паттерны:** есть ли повторяющиеся действия которые стоит зафиксировать?

## Шаг 2: Сформулировать уроки

Для каждого вывода определи:
- **Текст урока:** конкретное описание проблемы и правильного подхода
- **Категория:** в какую категорию Knowledge Base записать
- **Проект-источник:** в каком проекте это произошло

## Шаг 3: Записать в Knowledge Base (глобально)

> **НАСТРОЙКА:** замени `YOUR_VAULT_PATH` на путь к твоему Obsidian vault.
> Пример: `C:/Users/username/Documents/My Vault/Knowledge Base/`

**Путь к Knowledge Base:** `YOUR_VAULT_PATH/Knowledge Base/`

### 3.1: Определить категорию

1. Прочитать `_master-index.md` → список существующих категорий
2. Урок относится к существующей категории?
   - **ДА:** перейти к 3.2
   - **НЕТ:** создать новую категорию (папка + `_index.md`)
3. Категория = технология или домен (supabase, nextjs, design, deployment)
   - НЕ по типу ошибки (не "common-mistakes")
   - НЕ по проекту (не "my-site-lessons")

### 3.2: Проверить дубли

1. Прочитать `_index.md` целевой категории
2. Есть ли статья с похожей темой?
   - **ДА:** дополнить существующий файл новой секцией
   - **НЕТ:** создать новый файл

### 3.3: Записать урок

Формат нового файла:

```markdown
---
tags: [knowledge-base, КАТЕГОРИЯ]
source-project: ИМЯ_ПРОЕКТА
date: YYYY-MM-DD
---

# Заголовок урока

**Контекст:** что делал, когда столкнулся с этим.

**Проблема:** что пошло не так или что было неоптимально.

**Урок:** что делать правильно. Конкретно, actionable.

**Связанные:**
- [[Knowledge Base/КАТЕГОРИЯ/_index]]
```

Имя файла: kebab-case, описательное (`rls-service-role-key.md`, `isr-dynamic-rendering.md`).

Если дополняешь существующий файл:

```markdown

---

## Дополнение (YYYY-MM-DD, проект: ИМЯ)

**Контекст:** ...

**Урок:** ...
```

### 3.4: Обновить индексы

- Добавить строку в `_index.md` категории
- Обновить счётчик статей в `_master-index.md`

## Шаг 4: Lab-note в CLAUDE.md проекта

Краткая версия урока для быстрого контекста внутри проекта.

1. Найти CLAUDE.md в корне текущего проекта
2. Дописать в секцию `## Lab Notes`:

```
<!-- lab-note: YYYY-MM-DD описание проблемы -> правильный подход -->
```

## Шаг 5: Отчёт

Кратко (3-5 строк): сколько уроков, в какие категории, были ли новые категории.

## Правила

- НЕ писать очевидные вещи ("нужно читать файлы перед редактированием")
- НЕ писать общие советы ("быть внимательнее")
- Только конкретные, actionable выводы из ЭТОЙ сессии
- Если сессия прошла чисто: "Сессия прошла без значимых потерь"
- Максимум 5 уроков за раз. Лучше 2-3 самых важных
- Один урок = одна конкретная тема
