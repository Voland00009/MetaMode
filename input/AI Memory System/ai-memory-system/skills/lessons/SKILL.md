# /lessons — Загрузка уроков из Knowledge Base

Загружает релевантные уроки перед началом работы. Цель: не повторять ошибки, которые уже были.

## Синтаксис

```
/lessons                    — показать все категории
/lessons design             — загрузить все уроки по дизайну
/lessons supabase rls       — уроки по Supabase, фокус на RLS
/lessons всё                — загрузить ВСЕ уроки
```

## Что делать при вызове

> **НАСТРОЙКА:** замени `YOUR_VAULT_PATH` на путь к твоему Obsidian vault.

**Путь к Knowledge Base:** `YOUR_VAULT_PATH/Knowledge Base/`

### Без аргументов

1. Прочитать `_master-index.md`
2. Показать список категорий с количеством статей
3. Спросить: "Какую категорию загрузить?"

### С категорией (/lessons design)

1. Найти категорию в `_master-index.md`
2. Прочитать `_index.md` категории
3. Прочитать ВСЕ статьи
4. Вывести краткую сводку:

```
Загружено N уроков из категории "Design":
- design-orchestrator-always: всегда использовать перед UI
- shadcn-customization: semantic tokens вместо прямых цветов
Держу в контексте. Начинай задачу.
```

### С категорией + фокус (/lessons supabase rls)

1. Загрузить всю категорию
2. Выделить статьи с ключевым словом
3. Если совпадений нет: сказать об этом

### Всё (/lessons всё)

1. Для каждой категории: прочитать _index.md + все статьи
2. Если статей > 20: предупредить о нагрузке на контекст

## Правила

- НЕ копировать полный текст статей в ответ. Только краткую сводку
- Полный текст прочитать и ДЕРЖАТЬ В КОНТЕКСТЕ для использования при работе
- Если база пуста: "Knowledge Base пуста. Используй /reflect после сессий."
