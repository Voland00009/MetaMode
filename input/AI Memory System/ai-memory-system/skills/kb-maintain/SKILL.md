# /kb-maintain — Обслуживание Knowledge Base

Проверяет Knowledge Base на дубли, устаревшие записи и рассинхрон индексов. Ничего не удаляет без подтверждения.

> **НАСТРОЙКА:** замени `YOUR_VAULT_PATH` на путь к твоему Obsidian vault.

**Путь к Knowledge Base:** `YOUR_VAULT_PATH/Knowledge Base/`

## Шаг 1: Полное сканирование

1. Прочитать `_master-index.md`
2. Для каждой категории:
   - Прочитать `_index.md`
   - Прочитать все статьи в папке
   - Сравнить реальные файлы с записями в `_index.md`

## Шаг 2: Проверки

### Дубли
Две статьи описывают одну и ту же проблему → предложить смержить.

### Устаревшее
Урок старше 3 месяцев и касается быстро меняющейся технологии → пометить "проверить актуальность".

### Категории-сироты
Категория с 0 или 1 статьёй → предложить объединить с ближайшей.

### Мёртвые ссылки
Wiki-ссылки `[[...]]` на несуществующие файлы → предложить удалить или создать.

### Рассинхрон индексов
- Файл есть в папке, но не в `_index.md` (или наоборот)
- Счётчик статей в `_master-index.md` не совпадает с реальным

## Шаг 3: Отчёт

```
## KB Maintenance Report (YYYY-MM-DD)

**Статистика:** N категорий, M статей

**Проблемы:**
- [ДУБЛЬ] "file-a.md" и "file-b.md" описывают одно и то же
- [УСТАРЕЛО] "old-api.md" (от 2025-01-15) — API мог измениться
- [РАССИНХРОН] "new-lesson.md" не указан в _index.md

**Без проблем:** ✓
```

## Шаг 4: Исправления

Для каждой проблемы: спросить "Исправить?" и действовать только после подтверждения.

## Правила

- Это аудит, не автоочистка. Всегда спрашивать перед изменениями
- Если проблем нет: "Knowledge Base в порядке."
