# Self-Improving Loop (система памяти агента)

## Принцип

CLAUDE.md — не статичный файл. Это живая система из четырёх слоёв:

1. **Knowledge Compression** — сжатое описание проекта, структуры, ключевых файлов. Агент рассуждает "с высоты птичьего полёта" вместо чтения каждого файла.
2. **Preferences & Conventions** — правила, которые не встроены нативно. Стек, стиль кода, форматирование.
3. **Declaration of Capabilities** — явный список того, что агент МОЖЕТ делать. Claude часто не знает о своих возможностях.
4. **Log of Failures & Successes** — что работало, что нет. Сужает пространство решений.

## Триггер: после завершения задачи

После каждой нетривиальной задачи агент задаёт себе вопрос:

**"Как я мог бы прийти к тому же результату быстрее и за меньше токенов?"**

Если ответ содержит конкретный вывод — записать через `/reflect`.

## Триггер: после ошибки

1. Исправить ошибку
2. Запустить `/reflect` для записи урока

## Knowledge Base (Obsidian)

Глобальная база знаний: `YOUR_VAULT_PATH/Knowledge Base/`

Структура: `_master-index.md` → категории → статьи с wiki-ссылками.

**Скиллы:**
- `/reflect` — записывает уроки в KB после сессии
- `/lessons [категория]` — загружает уроки перед работой
- `/kb-maintain` — обслуживание (дубли, устаревшее, мёртвые ссылки)

**Правило:** перед крупной задачей (новая страница, деплой, новый проект) предложить `/lessons [категория]`.

## Lab Notes в CLAUDE.md проекта

Каждый проект хранит краткие lab-notes прямо в своём CLAUDE.md:

```
<!-- lab-note: 2026-04-06 описание проблемы -> правильный подход -->
```

Это быстрый кэш. Полная версия урока — в Knowledge Base.
