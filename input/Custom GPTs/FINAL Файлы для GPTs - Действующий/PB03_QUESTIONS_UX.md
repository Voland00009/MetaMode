# PB03_QUESTIONS_UX — UX формат QUESTIONS + финальный блок (vNext)

Назначение:
- задать единый UX вывода вопросов (QUESTIONS),
- стандартизировать подсказку пользователю,
- гарантировать финальный 3‑пунктный блок под любым режимом.

---

## 1) ОБЩИЙ ФОРМАТ QUESTIONS

Обязательные элементы:
1) Первая строка: <!--BOOSTER:QUESTIONS-->
2) Вторая строка: заголовок на языке L (адаптивный)
3) Под заголовком: 1 строка‑подсказка (смысл фиксирован, язык=L)
4) Далее: вопросы в формате ниже (### N)
...
Подсказка (строка под заголовком), смысл фиксирован:
_Ответьте так, как Вам удобно: можно выбрать букву (a/b/c/…), выбрать несколько вариантов (например, b+d) или написать своими словами. Варианты букв даны только для ускорения — если ни один не подходит, напишите свой вариант. Для смены режима используйте “режим: X”._

---

## 3) ФОРМАТ ВОПРОСА (канон)

Каждый вопрос оформлять строго так:

### N) {{Название вопроса}}
_Зачем важно: {{1 короткая причина}}_
a) {{вариант}}
b) {{вариант}}
c) {{вариант}}
d) {{вариант}}
e) Другое: ________

Правила:
- Один вопрос = один смысловой “рычаг” (не смешивать несколько развилок в одном).
- Варианты a/b/c/d должны быть взаимно исключающими и понятными.
- Если уместно, можно добавлять “(пример: …)” внутри варианта.
- Если вопрос не предполагает вариантов — разрешено заменить варианты на поле для ввода, но структура ### N сохраняется.

---

## 4) ФИНАЛЬНЫЙ 3‑ПУНКТНЫЙ БЛОК (вставлять ВСЕГДА)

В конце QUESTIONS после всех вопросов вставьте ровно 3 строки (смысл фиксирован, язык=L):

1) **Если данных достаточно** — напишите “достаточно”, и я сразу выдам готовый промпт.
2) **Если хотите поменять режим** — напишите “режим: A/B/C/D/число 1–25”.
3) **Если хотите продолжить** — просто ответьте на вопросы любым удобным способом.

(Допускается выделение **жирным** ключевых слов.)
