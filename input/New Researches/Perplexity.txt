Mem0 (mem0ai) offers a free tier limited to 10K memories with basic features, suitable for single-user testing but potentially insufficient for long-term daily coding sessions across projects. Paid tiers start at Standard ($19/month) for higher limits and Pro ($249/month) for graph memory; self-hosted open-source version is free with no feature gates, though graph differs from Pro. Recent 2026 GitHub issues note rate limiting on downloads (e.g., 429 errors), but production feedback on Reddit/HN is sparse; no major 2025-2026 changes documented beyond tier expansions. [github](https://github.com/mem0ai/mem0/issues/4745)

**Open-Source/Self-Hosted Options**
- **LightRAG (HKUDS/LightRAG)**: GraphRAG architecture with KV/vector/graph storage (e.g., NanoVectorDB/NetworkX); runs on modest hardware like Ollama-local setups (e.g., 1 worker, embedding dims 1024). Steep learning curve due to config-heavy server (env vars, chunking params); overkill for <10k docs per 2025 issues citing scale optimization needs—no 2026 single-user production reports found. [dev](https://dev.to/aairom/hands-on-experience-with-lightrag-3hje)
- **Graphiti (getzep/graphiti)**: Builds real-time temporal knowledge graphs for agents; operational burden noted as manageable for self-hosting but high if scaling (2025 feedback). No 2026 updates in releases; uncertain for single-user Claude Code without recent adoption data. [pub.towardsai](https://pub.towardsai.net/i-needed-memory-for-my-ai-agent-and-found-this-knowledge-graph-framework-55c94880820e)
- **Cognee**: Open-source knowledge graph memory with ECL pipeline (Extract/Cognify/Load) from 38+ sources; self-improving via feedback. Community repo active for plugins; suitable for single-user but no specific 2026 single-dev production feedback. [cognee](https://www.cognee.ai/blog/cognee-news/cognee-raises-seven-million-five-hundred-thousand-dollars-seed)
- **Letta (formerly MemGPT)**: Tiered OS-like memory (main/external storage) for unlimited context; 2026 positioning emphasizes local Letta Code app for personalized agents, model-portable, production-ready API—strong for single-user coding. [github](https://github.com/letta-ai/letta/issues/2444)

**Markdown/File-Based for Claude Code**
- **claude-mem**: GitHub/npm tool (thedotmack/claude-mem) for persistent memory via ChromaDB/MCP; local-only (~/.claude-mem), free forever, semantic search across sessions—install via npm. [github](https://github.com/hesreallyhim/awesome-claude-code/issues/167)
- **coleam00/claude-memory-compiler**: No direct repo found; uncertain existence or 2026 status—may reference Karpathy-inspired compilation patterns, but no recent GitHub hits.
- **Obsidian as storage**: Agent Client plugin integrates Claude Code CLI into Obsidian via MCP/tools; vault-aware (@note refs), but memory via external MCP servers rather than native—2025 forum use cases. [forum.obsidian](https://forum.obsidian.md/t/new-plugin-agent-client-bring-claude-code-codex-gemini-cli-inside-obsidian/108448)
- **Compile-on-idle via SDK**: Claude Code headless/Agent SDK (CLI/Python/TS) supports programmatic runs for idle compilation; no specific public GitHub "compile-on-idle" projects found, but hooks enable PreCompact/Stop triggers. [code.claude](https://code.claude.com/docs/en/headless)

**NotebookLM Integration**
No public API in 2026; forum confirms absence, with Vertex AI alternatives for agents but not direct backend. Unsuitable for automated pipelines due to no API; failure modes unknown without integration support. [discuss.ai.google](https://discuss.ai.google.dev/t/how-to-access-notebooklm-via-api/5084)

**New Entrants 2026**
Supermemory claims SOTA for single-user agent memory: graph-indexed, sub-300ms recall, solves long-term forgetting (LongMemEval_s benchmark); API for personalized LLMs, on-prem options—emerging but pricing/reviews sparse. [supermemory](https://supermemory.ai/research/)

**Claude Code Hooks**
Exposes: SessionStart, SessionEnd, Stop, StopFailure, PreCompact, PostCompact, UserPromptSubmit, PostToolUse, PostToolUseFailure, SubagentStart, SubagentStop, Notification (plus PreToolUse, PermissionRequest, TaskCreated). For text-only "save this moment": UserPromptSubmit/PreCompact hooks for manual triggers (e.g., shell/HTTP to dump context); patterns include transcript backup on Stop/PreCompact—no voice/mobile specifics documented. [mintlify](https://www.mintlify.com/instructkr/claude-code/advanced/hooks)
